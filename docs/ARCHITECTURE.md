# Architecture

## Overview

`moiskhan.dev` is a single Vite + React application that acts as both a portfolio and a project hosting shell. Individual projects live in **separate repos** and are deployed independently. The portfolio embeds them via `<iframe>` at `/projects/:slug`.

---

## Routing Structure

```
moiskhan.dev/                        → Portfolio landing page
moiskhan.dev/projects                → Projects showcase grid
moiskhan.dev/projects/:slug          → Live project embed (iframe)
moiskhan.dev/blog                    → Blog post listing
moiskhan.dev/blog/:slug              → Individual blog post
moiskhan.dev/achievements            → Hackathons & competition wins (also on homepage)
```

All routes are handled by **TanStack Router** using file-based routing under `src/routes/`.

## Data Flow

### Adding a new project (no component changes needed)

1. Deploy your project to its own URL (e.g., Vercel/Netlify)
2. Add one entry to `src/data/projects.ts`
3. It automatically appears in the `/projects` grid and gets a route at `/projects/:slug`

### Project embed mechanism

`/projects/:slug` reads the slug, looks up the project in `projects.ts`, gets the `liveUrl`, and renders it inside a full-viewport `<iframe>`. A thin overlay header shows the project name and a back button.

> ⚠️ Your deployed projects must NOT set `X-Frame-Options: DENY` or `Content-Security-Policy: frame-ancestors 'none'`. Most Vercel/Netlify deploys allow iframes by default.
